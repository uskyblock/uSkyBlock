# Coding Guidelines

This page is the new home for coding standards and pull request expectations.

## Current status

Guidelines are being migrated from the legacy wiki and will be updated for the current Gradle/Java 21 project setup.

## Baseline expectations

- Keep changes focused on one problem per pull request.
- Add tests when behavior changes.
- Avoid breaking public APIs without clear migration notes.
- Keep user-facing messages and configs backward-compatible when practical.
- Use clear commit messages and describe operational impact in the pull request.
